Rhino Linux 2023.4 - Enhancing the Experience, and looking towards 2024

After a long development period we're ready to finish off the year with a stellar release, as well as inform you, the community, of what to expect in 2024.

This development period was substantially longer than our previous releases, this can be attributed to the start of a new semester. If you wish to contribute to Rhino Linux and help us push out amazing upgrades faster, consider joining either our Matrix or Discord.

Release notes:

To upgrade to 2023.4 please run: rpk update -y.

The Unicorn Desktop

• Unicorn #39 has finally been merged, bringing optional auto-tiling to the Unicorn Desktop. Kick start your tiling window experience now, on start up you will notice an applet in your panel representing whether you are using our floating or tiling window mode.

• uLauncher now appears more rounded, and has a slightly different background colour.

rhino-pkg

• Changes in Nala v0.14.0 have affected how upgrades are handled. As a result, rhino-pkg now passes sudo nala upgrade --full --no-autoremove as the default upgrade functionality, providing Nala is installed and enabled.

Pine64

• Modem stability fixes have been shipped to both the PinePhone and the PinePhone Pro, as well as GPS support enablement

• The PineTab2 now features a more user-friendly experimental WiFi module enablement.

• All PinePhones and PineTabs now offer flashlight support.

All of these changes were brought over directly from Oren Klopfer's Ubuntu Touch port.

Other changes

• We have managed to close an open issue, where bluetooth support would be hit-and-miss on certain devices, this has now been resolved.

• A bug with our auto-tiling implementation (utilising cortile) and conky has been resolved.

• The "Your System" application now displays the version number as a pill, and can be copied to the clipboard on click.

• Pacstall 4.3.0 released, meaning that packages can now be tagged as "essential", and as such cannot be removed from the system. The rhino-core package, and its varients are now tagged as essential, meaning they’re assumed unremovable once installed. This will also mean unicorn is not automatically removed when switching between -cores, this must be a deliberate action. This makes Rhino Linux more stable in the long-term. If you wish to remove the Unicorn Desktop environment, you first must install rhino-server-core or rhino-ubxi-core, remove unicorn-desktop, and then install your preferred environment.

• pacstall-qa is now introduced as a new default package. pacstall-qa is a tool developed by Pacstall as a tool to test pascripts from PR's locally. You can read more about pacstall-qa here.

• During the 2023.4 development cycle we announced the launch of the The Unicorn Beyond XFCE initiative as a way to create official ports of Unicorn to other desktop environments.

• The installed kernels for this release are 6.6.7-generic for desktop, 6.7.0-rc5-okpine on Pine64 devices and 6.5.0-raspi for Raspberry Pi.

• The "Your System" application only displaying in light mode bug has been fixed.

• Connecting to WiFi on embedded devices has also been fixed.

What to expect in 2024

One of the larger things to expect in 2024, is the debut of our new icon pack! We've been hard at work designing an icon pack that is both visualy appealing whilst keeping the essence of the original artwork.

As we mentioned before about the Unicorn Beyond XFCE Initiative, in 2024 we will be aiding the development and creation of UBXI ports.

Rhino-pkg will be getting a full overhaul in nushell.

Pacstall 5.0.0 should release in 2024, which will bring significantly closer pacscript parity to PKGBUILDs.

There are also prospects of Unicorn becoming available on Wayland in 2024.

Some other things to expect in 2024 would be support for offline installations, as well as improved documentation on our Rhino Linux wiki.

• Many thanks, and happy holidays, from the Rhino Linux team.
