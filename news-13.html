Rhino Linux 2024.1

After a substantially longer than anticipated development period, we're happy to provide our first release of the year, as well as provide a tiny update on how the discussions about structuring Rhino Linux for development is going.

So before we get into information about this release, let's give a quick update. As you may have heard a while ago, we began the long discussions about how we wish to structure Rhino Linux going forward, after developer burnout became evident and development stagnated. We are happy to announce that after many discussions, we have currently drafted a working Rhino Linux constitution, with a hard focus on community involvement. These structural changes will begin to come into effect shortly after this release, so stay tuned for the full announcement!

If you wish to be a part of our upcoming community-led initiative, then consider joining us on Our Discord

Release notes:

To upgrade to 2024.1 please run: rpk update -y.

Known issues:

To give a little clarity, aside from a stagnation in development due to burnout, another reason as to why this release took so long was because everything that could have possibly gone wrong, did go wrong. These critical errors have thankfully been fixed, but some issues have persisted, we felt it more pertinent to release this update and make you (the community) aware of these issues than to wait any longer.

• On the liveboot, desktop icons are currently broken. To launch the Rhino Linux installer graphically you can use our application grid instead.

Pacstall:

Pacstall has now released v5.0.0, this is a major update which focuses primarily on quality of life features, enhanced security and functionality. Furthermore is now brings near total PKGBUILD parity. That's right, Pacscripts are now essentially no different to a PKGBUILD, meaning you can easily port packages from the AUR into Pacstall.

With bwrap being introduced in Pacstall 5.0.0, potentially dangerous scripts are now locked down by giving the child processes read-only access to every directory except the $PACDIR source directory and the staging directory. This will directory and the pkgdir staging directory. This will also kill any child processes after bwrap is finished, meaning that no extra code could possibly be left running after.

Now in Pacstall 5.0.0 you can also optionally specify version constraints and alternate dependencies on packages.

• You can read the full Pacstall 5.0.0 changelog here.

• You are also able to read the Pacstall 5.1.0 changelog here.

Other new features

• The installed kernels for this release are 6.9.1-generic for desktop, 6.9.0-okpine on Pine64 devices and 6.8.0-raspi for Raspberry pi.

• Package updates and small bug fixes.

• Many thanks from the Rhino Linux team.
